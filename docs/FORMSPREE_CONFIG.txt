----------------------------------------
WHY NOT @formspree/react?
----------------------------------------
The @formspree/react package wraps useState + fetch into a `useForm` hook with `state.submitting`,
`state.succeeded`, and `state.errors`. We already implement the equivalent (isSubmitting, sent,
submitError) by hand. Switching would:
  - Add a runtime dependency
  - Replace our nicer "Inquiry received." success card with a default <p>Thanks!</p> pattern
  - Lose the custom error banner and form reset logic
The raw-fetch approach is fewer lines, zero deps, and equivalent behavior. Stay put.
